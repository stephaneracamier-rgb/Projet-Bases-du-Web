Concepts clés

Partie 1 - Découvrez le web et ses origines

Web

Définition :
Comme une toile d'araignée, avec ses nombreux fils connectés, le web est un ensemble de pages, accessibles depuis un lien hypertexte.

Le lien hypertexte est probablement la plus grande invention à la base du Web. Le fait d'être sur une page et de cliquer sur un mot, ou même une image, pour en voir une autre paraît peut-être basique aujourd'hui, mais c'est la base d'une vraie révolution !
Voici celui de la page d'accueil de Google : www.google.fr. Le « www » présent dans les adresses signifie World Wide Web, soit « toile mondiale ».

Grâce aux liens hypertexte, chaque page sur le web est liée à d'autres pages. Par exemple, la page d'accueil d'OpenClassrooms fait des liens vers des cours, vers la page de connexion aux comptes, etc. De même, quand on fait une recherche sur Google, quel est le résultat ? Des liens vers plein de sites !

Un autre exemple : Wikipédia. Quand on consulte une page Wikipédia, il est fréquent d'avoir des liens vers d'autres articles, qui eux-mêmes pointent vers d'autres articles et ainsi de suite.

Internet

Définition :
Internet est un réseau informatique qui permet de communiquer entre ordinateurs dans le monde entier.
Sur ce réseau, l'on trouve plusieurs types de services :

  • Le Web (le plus connu d'entre eux) : pour ouvrir un navigateur web pour y aller, comme Google Chrome, Firefox, Internet Explorer, Edge, Safari, etc.
  • Les e-mails : pour échanger des messages. Il s'agit tout simplement du courrier électronique.
  • Les newsgroups : moins connus, ils sont l'ancêtre des forums et permettent de discuter à plusieurs en postant des messages.
  • Le FTP : un moyen d'échanger des fichiers entre ordinateurs.
  • Etc.

L'on confond souvent le Web avec le reste, car tous ces services convergent vers le Web. Par exemple auparavant, l'on utilisait obligatoirement un logiciel dédié pour les e-mails (Mozilla Thunderbird, Apple Mail, Outlook, etc.). Si ces logiciels existent toujours, aujourd'hui, l'on passe par le Web pour accéder à ses e-mails en allant par exemple sur le site de Gmail ou sur celui de La Poste.

Le Web sert donc de porte d'entrée à la plupart des services Internet aujourd'hui.

Le Cloud

Définition :
On parle souvent de Cloud... qu'est-ce que cela signifie concrètement ? Ce n'est pas évident, car c'est un terme (très) utilisé en marketing. Voyons voir !

Comme pour le mot Web, « Cloud » est un mot en anglais qui signifie « nuage ». Ceci donne un indice : imaginons un nuage dans le ciel, capable de maintenir de l'eau et de la relâcher sous forme de pluie à n'importe quel endroit.

Le « cloud » informatique opère de façon similaire, mais il conserve plutôt les données et services. Prenons l'exemple de photos personnelles. Lorsqu'elles sont stockées dans le cloud, cela évite de les garder en local sur son ordinateur. Le cloud est accessible de partout avec une connexion internet et s'adapte à tous les besoins : en cas de besoin de plus d'espace de stockage, il s'agrandit, évoquant ainsi la forme changeante d'un nuage.

Le cloud est composé des fameux services fournis via une interface web dont nous parlions ci-dessus. Par exemple, on l'pourrait dire que Gmail est un service d'email sur le Cloud.

L'on utilise chaque jour des services cloud :

  • Des services d'email : Gmail, Outlook, etc.
  • Des services de stockage de documents : Dropbox, Google Drive par exemple ;
  • Des services de messagerie : Slack, Discord, Messenger ;
  • Etc.

Le Web sert alors de passerelle à de nombreux services, sous la dénomination « cloud ».

W3C

Définition :

Création en 1994 du World Wide Web Consortium ou W3C, un organisme qui a pris le relais de Tim Berners-Lee pour faire évoluer les technologies du web (HTML, CSS, PNG, XML et autres noms barbares).

Partie 2 - Distinguez les langages du Web

Langages client

Définition

Les langages client servent à fournir des informations de base pour la construction des sites web. C'est un peu comme des plans d'architecte. Ces langages sont principalement :

  • HTML, qui permet de renseigner le contenu du site (textes ou images...) ;
  • CSS, qui permet de définir le visuel du site (les couleurs, la police, les tailles...);
  • JavaScript, qui permet d'ajouter de l'interaction sur le site (menus déroulants, affichage de messages lors de la validation de formulaires...).

Ces trois langages sont désormais indispensables à la réalisation de tous les sites web. On dit que ce sont des langages client ou encore des langages frontend, car ils sont lus par les machines des clients à travers les navigateurs web.

Nous n'apprendrons pas à nous en servir dans ce cours, mais vous pourrez enchaîner avec d'autres cours accessibles aux débutants si vous voulez vous entraîner à les utiliser !

Navigateurs

Définition

Pour accéder aux sites web, on a besoin de navigateurs web. Il existe notamment :

  • Google Chrome
  • Mozilla Firefox
  • Edge (le successeur d'Internet Explorer)
  • Safari

Le rôle des navigateurs est de traduire les langages HTML, CSS et JavaScript sous la forme de sites web utilisables par tout le monde.

Langages serveur

Définition

Les langages serveur sont, comme leur nom l'indique, gérés par les serveurs. Les clients n'y touchent pas. Leur rôle est un peu différent :

  • Les langages client décrivent comment le site web doit s'afficher.
  • Les langages serveur décrivent comment le site web doit se comporter.

Subtil, mais important !

  • Avec un langage client, je peux dire « Un menu doit s'afficher à gauche de mon site web ».
  • Avec un langage serveur, je peux dire « Le menu ne doit s'afficher que si cette personne a créé un compte sur mon site ».

De ce fait, tous les langages communiquent entre eux pour fournir un résultat lisible côté client !

Les langages serveur sont nombreux. En voici quelques exemples :

  • PHP
  • Java (rien à voir avec JavaScript, attention !)
  • Python
  • C#
  • ...

On trouve tous ces langages parmi les sites web les plus connus. On trouve par exemple du PHP chez Facebook et OpenClassrooms, du Java sur le site de votre banque, du Python et du Java chez Google, etc.

Frameworks

Définition

Il est nécessaire au préalable de définir les bibilothèques ou librairies. Ce sont des morceaux de code écrits par d'autres développeurs qui sont réutilisables pour gagner du temps. On les utilise souvent pour des besoins courants, comme pour créer des aspects interactifs d'un site. Mais on peut aussi les utiliser pour des besoins spécifiques, qui nous prendraient d'énormes heures et beaucoup d'efforts et de compétences pour coder de 0. JQuery pour le langage JavaScript est un exemple d'une bibliothèque.

Les frameworks sont des bibliothèques ou librairies très élaborées qui permettent de structurer l'entièreté de son code, comme un kit de construction ! Ils sont utilisés notamment pour des sites web plus importants, qui nécessitent beaucoup de code. Vous avez peut-être déjà entendu parler d'Angular pour JavaScript ? Il s'agit d'un framework assez connu. Pour utiliser des frameworks vous devez les télécharger et les inclure dans les projets sur votre IDE.

Bases de données

Définition

Tous les sites web ont besoin d'enregistrer des informations, comme la liste de leurs utilisateurs, des messages qui ont été échangés, etc. C'est là qu'une base de données intervient : c'est un logiciel dédié au stockage de données.

Parmi les logiciels de base de données les plus célèbres, on peut citer :

  • MySQL
  • PostgreSQL
  • SQL Server
  • Oracle
  • SQLite
  • Etc.

Pour communiquer avec ces logiciels, on utilise un langage : SQL.

On dit que l'on fait des requêtes SQL. Par exemple : « Je veux la liste des derniers utilisateurs inscrits sur mon site » ou « Je veux stocker ce message ».

Site responsive

Définition

Un site web responsive s'adapte automatiquement à la petite taille des smartphones et à la taille des tablettes.
Les sites web responsive sont développés avec les langages du web suivants : HTML, CSS, JavaScript... et d'autres langages de développement.

Application mobile

Définition

Une application « native » est conçue spécialement et uniquement pour un smartphone ou une tablette. Cette option est lourde mais elle offre une très bonne expérience pour un service complexe.
L'application se développe avec des langages mobiles spécifiques à Android et IOS (par exemple : Kotlin, Java ou Swift.)

Partie 3 - Découvrez le fonctionnement du réseau

Serveurs et datacenters

Définition

L'on se représente souvent une toile d'araignée pour désigner le Web. En effet, deux types d'ordinateurs communiquent entre eux via des nœuds.

Les serveurs sont les machines (ordinateurs) qui hébergent les sites web et qui permettent aux machines clientes (ordinateurs des internautes) d'y accéder.
Plats et sans écran, les serveurs sont empilés les uns sur les autres dans des baies informatiques et sont connectés via des câbles réseaux.

Les datacenters sont des bâtiments très sécurisés dotés de salles blanches dans lesquelles sont hébergés de nombreuses baies de serveurs.
Les datacenters reliés à un câble de fibre optique permettant un échange ultrarapide. On retrouve cette fibre souvent enterrée sous terre, mais aussi... au fond de la mer ! En fait, la majorité du trafic d'Internet passe par des câbles sous-marins.

Les serveurs locaux ne sont pas de véritables serveurs car ils présentent une double exception. En effet ils ne sont pas hébergés dans des datacenters et ce ne sont pas de véritables serveurs mais ce sont les ordinateurs des développeurs.
Les serveurs locaux permettent aux développeurs de créer et tester en langage backend (PHP, Node.JS, Python, etc.) des sites Web sur leurs propres ordinateurs grâce à des outils permettant de simuler le fonctionnement d'un serveur.
On peut ainsi exécuter les langages backend, sans avoir besoin de mettre son site sur un vrai serveur pendant la phase de développement. Les plus connus sont MAMP pour Mac, WampServer pour Windows ou encore LAMP pour Linux.

Adresse IP

Définition

Chaque ordinateur possède une adresse, on parle alors d'adresse IP. Il s'agit d'une suite de nombres comme 205.89.177.26. Vous pouvez voir cela comme une sorte de numéro de téléphone.

En théorie, vous pouvez donc aller sur un site web en tapant directement l'adresse du serveur dans votre barre d'adresse. Mais cela n'est pas très pratique, c'est pourquoi ont été créés les noms d'hôtes tel « google.com ».

Nom d'hôte et DNS

Définition

Les noms d'hôte comme « google.fr » permettent de se rappeler très facilement du nom du service (site Internet) que l'on veut contacter.

Et l'on a ensuite créé un service d'annuaire, les DNS (Domain Name Service), pour faire le lien entre le nom d'hôte et l'adresse IP.

Le DNS permet de traduire le nom d'hôte en adresse IP. On peut le voir comme un annuaire géant où l'on retrouve tous les noms d'hôtes (google.fr, openclassrooms.com, etc) avec l'adresse IP associée.

Protocoles

Définition

Les protocoles sont des langages de communication pour les ordinateurs et les serveurs. Ces langages permettent aux ordinateurs de dialoguer selons les protocoles dits de bas niveau et de haut niveau.

Les protocoles de bas niveau sont à la base d'Internet, on les retrouve dans toutes les communications :

  • TCP (Transmission Control Protocol)
  • UDP (User Datagram Protocol)

TCP en particulier est très important. Il a été inventé par Vint Cerf, considéré aujourd'hui grâce à cela comme le père d'Internet. Sans TCP, pas de communication réseau... et Tim Berners-Lee n'aurait jamais pu inventer le Web.
TCP est utilisé pour un peu tout : faire transiter des pages web, des e-mails, des vidéos...

Les protocoles de haut niveau sont généralement basés sur TCP (ou UDP). Ils forment une surcouche qui englobe le protocole TCP.

Les noms des protocoles haut niveau devraient vous être un peu plus familiers pour certains :

  • HTTP : le protocole qui permet d'échanger des pages web entre le client et le serveur. En plus du langage HTML, Tim Berners-Lee a inventé les bases du protocole HTTP qui permet d'échanger les pages. C'est ce que signifie le "http://" que vous voyez au début des adresses web !
  • HTTPS : identique à HTTP, avec le "S" en plus qui signifie "Secure". Les pages sont chiffrées, pour garantir que personne ne peut les lire (par exemple quelqu'un qui espionnerait les données qui transitent dans les câbles sous-marins !).
  • FTP : permet d'échanger des fichiers.
  • SMTP : permet d'envoyer des e-mails.
  • Etc.

Partie 4 - Identifiez les métiers du web

Développeurs

Définition

Le développeur est une personne qui conçoit, teste, documente et maintient des applications, des sites Internet ou des logiciels.

Contrairement à ce qu'on peut s'imaginer, le développeur ne passe pas toute sa journée à écrire des lignes de code. Quelle que soit sa spécialité, il passe beaucoup de temps à tester son code, à chercher à optimiser ses méthodes de développement, et à travailler avec les autres !
L'un de ses plus grands challenges est d'analyser le besoin initial du client ou de l'utilisateur : le développeur doit comprendre ce besoin pour le traduire en langage de programmation.

Voici quelques spécialités

  • le développeur front-end : le développeur front-end travaille sur toute la partie apparente du site, ce que nous voyons lorsque nous allons sur une page web. Il crée l'interface utilisateur de telle sorte qu'elle soit la plus adaptée au besoin, et la plus simple possible. Il utilise les langages client tels que HTML, CSS ou JavaScript ;
  • le développeur back-end : le développeur back-end travaille au contraire sur toutes les composantes cachées d'un site. Il crée l'infrastructure adaptée, et coordonne toutes les relations entre le site et les bases de données. Il connaît bien un ou plusieurs langages de programmation (PHP, Ruby, Python, Java) et le langage des bases de données (SQL), ainsi que les frameworks qui lui facilitent la tâche ;
  • le développeur mobile : il développe des applications Android ou iOS, parfois même les deux, même si c'est plus rare ! Il travaille sur toutes les étapes de la conception : depuis le cahier des charges jusqu'à la correction des bugs. Quand on développe sur mobile, on travaille en général aussi bien sur l'interface (front-end) que sur la logique (back-end) ;
  • le développeur low-code : ce développeur utilise des plateformes qui permettent de créer des applications avec peu ou pas de code. L'objectif est d'assembler des blocs fonctionnels existants pour construire rapidement des solutions. Le développeur low-code connaît les principes de la programmation et les besoins métier, mais il n'a pas forcément besoin d'écrire beaucoup de code lui-même. Ces outils sont très utilisés dans les entreprises pour créer des applications internes ou automatiser des processus sans mobiliser une équipe de développement complète.

Product Managers et chefs de projets

Définition

C'est pour éviter les désagréments de types : fonctionnalités désordonnées et problèmes de mises à jour, qu'existent les Product Owners et les Product Managers.
Le Product Owner veille à ce que chaque détail du produit réponde aux besoins immédiats des utilisateurs et soit correctement implémenté.
Le Product Manager, quant à lui, assure que le produit évolue dans la bonne direction, en anticipant les besoins futurs et en définissant une stratégie à long terme.
Ensemble, ils garantissent que le produit soit à la fois fonctionnel et en phase avec les attentes du marché.

Le quotidien du Product Owner
Très concrètement, le Product Owner doit s'assurer que le produit développé réponde aux besoins des utilisateurs et aux objectifs par le Product Manager. Pour réussir sa mission, le Product Owner va rédiger et prioriser les user stories, des descriptions courtes et précises des fonctionnalités à développer. Par exemple, il peut écrire une user story comme « En tant qu'utilisateur, je veux pouvoir filtrer les produits par prix afin de trouver rapidement ceux qui correspondent à mon budget. » Il anime les réunions de planification et de revue, s'assurant que chaque fonctionnalité livrée apporte de la valeur au produit.
Il teste régulièrement les développements pour vérifier qu'ils correspondent bien aux attentes initiales. Par exemple, avant de valider une nouvelle fonctionnalité de filtre sur un site e-commerce, il teste différentes combinaisons pour s'assurer que le filtre fonctionne correctement.
Enfin, le Product Owner travaille étroitement avec les développeurs, les designers et les testeurs, veillant à ce que la vision du produit soit bien comprise et respectée.

Le quotidien du Product Manager
Le Product Manager a une vision plus stratégique du produit. Il analyse le marché, identifie les opportunités et définit la roadmap produit à long terme. Pour réussir sa mission, le Product Manager mène des études de marché, analyse les données utilisateurs et surveille les tendances. Par exemple, il peut étudier les besoins des consommateurs pour lancer une nouvelle fonctionnalité de paiement mobile sur une application bancaire.
Il élabore une vision globale du produit et s'assure que toutes les parties prenantes partagent cette vision. Il collabore avec les équipes marketing pour définir les stratégies de lancement et avec les équipes de vente pour optimiser la commercialisation. Par exemple, avant le lancement d'un nouveau produit, il planifie une campagne publicitaire et forme l'équipe de vente sur les nouvelles fonctionnalités. Il est responsable de la rentabilité du produit et de son évolution à long terme, s'assurant que chaque décision prise contribue à la réussite du produit sur le marché.

Designers

Définition

L'UX Designer travaille à ce que l'internaute (ou le mobinaute) s'oriente aisément et trouve ce qu'il est venu chercher de manière efficace. Il fait en sorte que l'utilisation d'un site ou d'une application soit la plus simple possible, que la navigation soit fluide et intuitive ; bref, qu'on prenne plaisir à utiliser le produit.
Très concrètement, l'UX Designer doit s'assurer que le site ou l'application réponde aux attentes et besoins de ses utilisateurs, qu'il procure des émotions positives.

Pour réussir sa mission, l'UX Designer va utiliser plusieurs méthodes “centrées utilisateur”. Il va définir le public cible du produit, mener des “recherches utilisateur” : via des entretiens, des observations in situ, des analyses de données, il cherche à comprendre et à analyser les attentes des utilisateurs. Il va créer des prototypes, les tester avec de potentiels futurs utilisateurs... C'est ce qu'on appelle la conception centrée sur l'humain ou Human-Centered Design (HCD).
Le quotidien d'un UX Designer est donc fait de beaucoup d'interactions : l'UX Designer rencontre régulièrement des utilisateurs et doit faire preuve d'empathie pour comprendre leurs besoins et attentes. Aussi, l'UX Designer est en lien avec les équipes techniques, dont les développeurs ! Ces professionnels travaillent tous pour concevoir le produit, mais sous des angles différents et complémentaires.

Quelle différence entre l'UX et l'UI Designer ?

Concrètement, l'UI Designer mobilise des méthodologies très proches de l'UX, centrées sur l'humain. Alors que l'UX Designer s'intéresse à l'expérience globale de l'utilisateur, l'UI Designer va se concentrer sur l'interface. En lien avec les équipes techniques, il donne vie au produit en mixant des éléments graphiques et des éléments fonctionnels, de telle sorte que le produit soit le plus attractif, agréable et accessible possible.

En résumé, l'UI Designer et l'UX Designer travaillent sur des missions qui se recoupent. D'ailleurs, dans certaines structures, on ne différencie pas les deux métiers et on parle de “Product Designer”. C'est souvent le cas dans les petites entreprises, qui vont chercher des profils généralistes capables de travailler sur toutes les dimensions du design d'un produit.

Testeurs

Définition

La Quality Assurance, ou QA, est le nom que porte l'équipe en charge de s'assurer que les produits développés sont fiables, fonctionnent correctement et offrent une expérience utilisateur optimale.

L'équipe QA utilise des processus de test rigoureux, qui comprennent souvent des tests manuels et automatisés, pour identifier les défauts et bugs dans les sites ou applications web. En collaborant avec les équipes de développement, l'équipe QA contribue à améliorer la qualité du produit, à réduire les risques et à renforcer la confiance des clients.

Les métiers principaux au sein de l'équipe QA comprennent entre autres :

  • L'ingénieur QA, ou le testeur logiciel : Il est chargé de concevoir, d'exécuter et d'automatiser les tests pour garantir la qualité des logiciels ou des produits. Il identifie les bugs et les problèmes de performance afin de les résoudre avant la mise en production.
  • L'analyste QA : il analyse les exigences des clients et les spécifications des produits pour élaborer des plans de test détaillés. Il s'assure que les fonctionnalités développées correspondent aux attentes et aux besoins des utilisateurs.
  • Le responsable QA : Il supervise et coordonne les activités de l'équipe QA. Il définit les stratégies de test, établit les normes de qualité et assure la conformité aux processus et aux exigences de l'entreprise.

Techniciens informatique

Définition

Le rôle du technicien informatique au sein d'une entreprise est d'assurer le bon fonctionnement du parc informatique, et de fournir aux utilisateurs des solutions pour pallier les éventuelles pannes ou difficultés qu'ils peuvent rencontrer.

Le poste de technicien informatique est extrêmement polyvalent ! Voici notamment ce qu'il est amené à faire :

  • Il supervise le matériel informatique, tels que les ordinateurs et les périphériques, ainsi que les systèmes d'exploitation (Windows, OS X, Linux...). Il configure et entretient le parc informatique, gère les stocks et veille à son bon fonctionnement.
  • Il s'assure que tous les postes de travail aient accès aux applications et services nécessaires, en mettant en place l'architecture des câbles réseaux et du WiFi.
  • Le technicien informatique assure le support et le conseil aux utilisateurs, en étant leur interlocuteur privilégié pour tous les problèmes informatiques, de la réception des demandes jusqu'à l'assistance en cas de panne. Il gère les problèmes en fonction de leur urgence et aide les utilisateurs à devenir autonomes. C'est un métier qui exige patience et pédagogie.

Langages de développement web

HTML

langage Front-End

Squelette du site Web. Il définit la structure de base du contenu, comme les titres, les paragraphes, les images, etc.

CSS

Langage Front-End

Graphisme du site Web. IL permet de définir l'aspect visuel dudit site : les couleurs, la police, les tailles, etc.

JavaScript

Langage Front-End

Facilitateur d'interaction pour le site Web. Il permet d'ajouter des menus déroulants, de l'affichage de messages lors de la validation de formulaires, etc.

JavaScript

Langage Back-End

Gestionnaire du serveur et de l'interface Web. Il permet de gérer des applications de chat en temps réel, du streaming vidéo, des infrastructure backends, etc.

PHP

Langage Back-End

Exécuteur de requêtes en langage PHP. Il permet de créer des pages dynamiques et interactives, d'accèder aux bases de données, de gérer des sessions utilisateur, etc.

Java

Langage Back-End

Langage multiplateformes orienté objet. Il permet de développer des applications mobiles, des API, des jeux vidéo, des technologies côté serveur, etc.

Python

Langage Back-End

Langage de programmation interprétative structurée. Il facilite l'apprentissage automatique, il permet de créer des sites Internet, des tests logiciels, etc.

C#

Langage Back-End

Langage de programmation orienté objet. Il permet de créer des sites Web et des ERP, de réaliser du scripting, de développer des applications lourdes, etc,

SQL

Langage pour bases de données

Exécuteur de requêtes SQL. Il permet d'interroger les bases de données relationnelles et d'assurer la définition, la gestion et le contrôle des données.

Boîte à outils du développeur

  • IDE (Integrated Development Environment) :

    Quand l'on souhaite écrire un document, on utilise un logiciel de traitement de texte (Word, OpenOffice, LibreOffice, etc.).
    Quand l'on veut écrire du code, l'on utilise soit un éditeur de texte) avec des fonctionnalités étendues (Notepad++, Sublime Text, etc.), soit comme un IDE ou Integrated Development Environment.

    Pour travailler sur un environnement de développement très efficace et disposant de nombreuses fonctionnalités, il est nécessaire d'utiliser un IDE qui signifie en française : « environnement de développement intégré ».
    Les IDE les plus connus sont Visual Studio Code (VSCode), JetBrains IntelliJ IDEA, Eclipse, NetBeans, etc.

  • Github :

    Github est un service en ligne d'hébergement de versions successives d'un fichier ou d'un code, établies avec le gestionnaire de versions Git.

    Logiciel de gestion de versions décentralisé, Git permet de travailler de manière collaborative sans risquer de détruire les modifications de code réalisées par d'autres développeurs.
    Github permet d'héberger les versions du code et des fichiers avec un système de suivi des modifications et de traçabilité des interventions de chaque développeur.

  • Librairies / bibliothèques :

    Une librairie ou blibliothèque est un ensemble de fonctions et/ou de classes déjà codées dans un langage spécifique. Le développeur va alors directement piocher le code nécessaire dans une librairie du langage de programmation utilisé.
    Les bibliothèques logicielles permettent de simplifier et de rationaliser la programmation. C'est également un gain de temps pour les développeurs, ce qui leur permet également d'améliorer la qualité de leur code tout en bénéficiant du soutien de la communauté de leurs pairs.

  • Frameworks :

    Egalement appelé infrastructure logicielle ou cadre d'applications, le framework est un ensemble de composants logiciels structurels, de bibliothèques, de modèles de conception, d'outils et de conventions de codage qui accèlèrent le processus de développement logiciel.
    Le framework se distingue d'une bibliothèque ou d'une librairie par son caractère beaucoup plus générique, il peut d'ailleurs compter plusieurs bibliothèques. Servant de cadre au développement de progiciels et autres applications, il en modèle l'architecture logicielle.

    Un framework peut être dédié à une plateforme informatique, à un domaine spécifique ou à un langage de programmation spécifiques, exemples : Symfony pour PHP, Django pour JavaScript ou .Net pour C# et Visual Basic .NET.
