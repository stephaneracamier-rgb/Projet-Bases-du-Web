Concepts clés

Partie 1 - Découvrez le web et ses origines

Web

Souvent assimilé à une toile d'araignée avec ses très nombreux fils, le web peut être décrit comme un ensemble de pages interconnectées via des liens hypertextes.

L'invention du lien hypertexte est probablement la contribution la plus importante à l'origine du Web. En effet le passage d'une page (un site Internet) avec un simple clic sur un lien se présentant sous forme d'un mot ou d'une image, paraît basique, mais cela a été une vraie révolution !
Voici le lien hypertexte vers la page d'accueil du moteur de recherche Google : www.google.fr. Le « www » présent dans les adresses des sites Internet signifie World Wide Web, que l'on peut traduire par « toile mondiale ».

Grâce aux liens hypertexte, chaque page sur le web est liée à d'autres pages. Ainsi sur la page d'accueil du site d'OpenClassrooms sont présents des liens vers des cours, vers la page de connexion aux comptes, etc. De même quand on fait une recherche sur Google.fr ou sur Google.com, l'on trouve de nombreux liens vers des sites Internet !

Le lien hypertexte vers le site de la très connue encyclopédie en ligne peut se présenter ainsi : Wikipédia. Quand on consulte l'une de ses pages, il est alors fréquent d'avoir des liens vers d'autres articles, qui eux-mêmes vont avoir des liens vers d'autres articles, etc.

Internet

Internet est un réseau informatique qui permet de faire communiquer entre eux des ordinateurs dans le monde entier.

Sur le réseau des réseaux, comme l'on nommeégalement Internet, l'on trouve plusieurs types de services :

  • Le Web (le plus connu d'entre eux) est constitué de sites ou de pages, auxquels on accède via un navigateur dédié comme Google Chrome, Firefox, Internet Explorer, Edge, Safari ou Opera.
  • Les e-mails permettent d'échanger des messages électroniques ou courriels. Il s'agit tout simplement de l'envoi et de la réception de courriers électroniques.
  • Les newsgroups constitue un service moins connu. Ils représentent les ancêtres des forums électroniques et permettent de discuter à plusieurs en postant des messages.
  • Le FTP ou File Transfer Protocol est un moyen facile d'échanger des fichiers de toutes tailles entre ordinateurs.
  • Etc.

L'on confond souvent le Web avec les autres services Internet, c'est parce qu'ils sont maintenant utilisables via le Web. Alors qu'auparavant l'on utilisait un logiciel de messagerie pour les e-mails (Mozilla Thunderbird, Apple Mail, Outlook, etc), maintenant l'on passe souvent par le Web pour accéder à ses e-mails (par exemple : www.gmail.com ou www.laposte.net).

Le Web sert donc de porte d'entrée à la plupart des services Internet aujourd'hui.

Cloud

L'on parle souvent de Cloud mais ce terme n'est pas clair pour tout le monde, car il est très utilisé en matière de marketing ou de manière impropre.

« Cloud » est un mot anglais qui se traduit par « nuage », car l'on peut le comparer à un nuage dans le ciel capable de stocker et de relâcher de l'eau, n'importe ou, sous forme de pluie.

Le « cloud » informatique fonctionne de façon similaire, mais il concerne plutôt les données et les services. Ainsi le stockage de photos personnelles dans le cloud permet d'éviter de les garder sur son propre ordinateur, le cloud étant accessible de partout avec une connexion internet.
Le Cloud s'adapte à tous types de besoins en matière de stockage de données ou de services en ligne. Lorsqu'un besoin supplémentaire est identifié, l'espace de stockage peut s'agrandir comme un nuage dont la forme ou la taille peuvent changer.

Le cloud est composé de services fournis et accessibles via une interface web. L'on pourrait ainsi dire que Gmail est un service d'email utilisable via le Cloud.

Les internautes utilisent chaque jour des services cloud sous les formes suivantes :

  • des services d'email comme Gmail, Outlook, Orange, Free, etc ;
  • des services de stockage de documents tels Dropbox, Google Drive, Microsoft OneDrive, OVH, etc ;
  • des services de transfert de fichiers lourds ainsi que WeTransfer, Smash, Transfernow, Send, etc ;
  • des services de messagerie pareils que Slack, WhatsApp, Discord, Messenger ou Telegram,
  • etc.

En conclusion : le Web sert donc de passerelle à de nombreux services, sous la dénomination de « cloud ».

W3C

Création en 1994 du World Wide Web Consortium ou W3C, une organisation non gouvernementale qui a pris le relais de Tim Berners-Lee afin de faire évoluer les technologies du web (HTML, CSS, PNG, XML, etc).
Le W3C est deevenu un organisme iternational de standardisation du Web rassemblant de très nombreux acteurs d'Internet issus plusieurs dizaines de pays : centres de recherche informatique, universités, éditeurs de logiciels, éditeurs de navigateurs Internet, opérateurs de réseaux de télécommunications, etc.

Le W3C établit les standards du Web sous la forme de spécifications techniques destinées à tous les acteurs du secteur : agences Web, hébergeurs de sites Web, éditeurs de logiciels SAAS (Software As A Service ou logiciel en tant que service) développeurs indépendants, éditeurs de CMS (Content Management System ou Système de Gestion de Contenus), etc).
Les Recommandations du W3C sont établies par des groupes de travail afin d'établir des normes d'accessibilité du Web pour tous appareils (ordinateurs, tablettes, smartphones, TV, consoles, etc) et pour tous publics, y comprit les personnes en situation de handicap (non voyants, mal voyants, personnes sourdes et/ou muettes, etc).

Partie 2 - Distinguez les langages du Web

Langages client

Les langages client servent à fournir des informations de base pour la réalisation de sites web. Par langage client il faut comprendre un langage informatique directemnt interprété par le navigateur Web de l'internaute afin de lui permettre de visualiser des contenus.

Trois langages clients sont à a base de la construction des sites Web :

  • Le HTML (HyperText Markup Langage) permet de renseigner les différents contenus du site : textes, images, etc.
  • Le CSS (Cascading Style Sheets) permet de définir les modalités d'affichage des contenus du site : couleurs des fonds, police d'écriture, tailles des encarts, etc.
  • Le JavaScript permet d'ajouter de l'interaction au site : menus déroulants, affichage de messages lors de la validation de formulaires, galerie d'images affichables en grand format, etc).

Ces trois langages sont désormais indispensables à la réalisation de tous types de sites web. L'on dit que ce sont des langages client ou encore des langages frontend, car ils sont lus par les machines des clients àu travers de leurs navigateurs web.

Navigateurs

Pour accéder aux sites web il est nécessaire d'utiliser un navigateur web. Les principaux navigateurs gratuits et pré-installés sur les appareils (ordinateurs, smartphones, etc) sont :

  • Google Chrome,
  • Mozilla Firefox,
  • Edge (le successeur d'Internet Explorer),
  • Safari,
  • Opera.

Le rôle des navigateurs est de traduire les langages HTML, CSS et JavaScript en sites web utilisables par toutes sortes d'utilisateurs.

Langages serveur

Les langages serveur sont, comme leur nom l'indique, gérés par les serveurs Web ; alors que les langages clients sont gérés par les navigateurs Web.

Pour l'internaute les rôles des deux types de langage sont les suivants :

  • Les langages client décrivent comment le site web doit s'afficher.
  • Les langages serveur décrivent comment le site web doit se comporter.

Pour mieux comprendre le fonctionnement des sites web, l'on peut établir avec l'exemple suivant le rôle de chacun d'entre eux :

  • avec un langage client « le menu doit s'afficher à gauche du site web » ;
  • avec un langage serveur « le menu ne doit s'afficher que si l'internaute a créé un compte sur le site ».

En conclusion, tous ces langages communiquent entre eux pour fournir un résultat lisible côté utilisateur !

Les langages serveur sont nombreux, en voici quelques exemples :

  • PHP,
  • Java (à ne surtout pas confondre avec le langage frontend JavaScript),
  • Python,
  • Ruby,
  • C#,
  • Etc.

L'on retrouve tous ces langages sur de très nombreux sites Internet, notamment pour les plus connus d'entre eux. L'on trouve par exemple du PHP surles sites de Facebook et d'OpenClassrooms, du Java sur les sites bancaires, du Python et du Java sur le site de Google.com, etc.

Frameworks

Au préalable, il est nécessaire de définir le concept de bibliothèques ou de librairies afin de mieux comprendre ce qu'est un Framework.

Les bibliothèques logicielles sont constituées d'un ensemble de routines informatiques qui ont déjà été écrites par d'autres développeurs et sont réutilisables en l'état, pour des besoins spécifiques ou des nécessités courantes.
L'utilisation de librairies, pour un développement informatique, permet d'économiser un grand nombre d'heures ainsi que beaucoup d'efforts et de compétences pour coder à partir de rien. Pour le langage JavaScript, JQuery est un exemple de bibliothèque, tandis que PHPMailer est une librairie pour le langage PHP.

L'on ne peut pas considérer les frameworks comme des librairies car ils comportent également de nombreux autres outils : modèles d'organisation du code, fichiers d'application, fonctionnalités courantes, documentation, etc.
Les frameworks permettent de structurer l'entièreté de son code et l'on peut les définiir comme un kit complet de développement logiciel ! Ils sont utilisés notamment pour des sites web importants qui nécessitent beaucoup de code, mais également pour développer d'autres types de logiciels.

Les frameworks les plus utilisés sont les suivants :

  • Angular et ReactJS pour le langage JavaScript ;
  • Django pour le langage Python ;
  • Ruby on Rails pour le langage Ruby ;
  • Symfony et Laravel pour la langage PHP.

Pour utiliser un framework il est nécessaire de le télécharger et de l'inclure dans les projets prsénts sur son propre IDE.

Bases de données

Tous les sites web ont besoin d'enregistrer des informations telles : la liste des utilisateurs, les contacts clients, les messages qui ont été échangés, etc.
C'est là qu'intrvient la base de données relationnelle intervient, c'est un logiciel dédié à l'enregistrement, au stockage et à l'utilisation des données.

Parmi les logiciels de création et gestion de bases de données les plus utilisés, l'on peut citer :

  • MySQL,
  • PostgreSQL,
  • SQL Server,
  • Oracle,
  • SQLite,
  • Etc.

Pour communiquer avec ces logiciels, l'on utilise un langage informatique spécifique nommé SQL (Structured Query Langage), traduit en français par langage de requêtes structurées).

L'on dit alors que l'on écrit ou éxécute des requêtes SQL, ce que l'on pourrait traduire par : « Je veux la liste des derniers utilisateurs inscrits sur mon site » ou « Je veux stocker ce message ».

Site responsive

Un site web responsive s'adapte automatiquement à la petite taille des différents modèles de smartphones ainsi qu'aux tailles des divers modèles de tablettes.
Les sites web responsive sont développés avec les langages web orientés client (HTML, CSS et JavaScript). Tandis que les langages orientés serveur (PHP, Java, ASP, etc), pour affcher des contenus, interagissent avec la base de données hébergées sur le serveur.

Application mobile

Une application « native » est conçue spécialement pour ue plateforme spécifique ou un dispositif particulier. Ainsi une application mobile est développée pour fonctionner sur un smartphone doté d'un système d'exploitation ioS ou Android
Installée directement sur le téléphone après chargement sur un magasin d'applications (Google Play, App Store ou Windows Store), l'application mobile est lourde mais elle offre une très bonne expérience pour un service complexe.

Les applications mobiles sont développées avec des langages mobiles spécifiques à Android ou à IOS que sont : Kotlin, Java, Swift, Objective-C, etc).

Partie 3 - Découvrez le fonctionnement du réseau

Serveurs et datacenters

L'on se représente souvent le Web comme une toîle d'araignée, avec deux types d'ordinateurs qui communiquent entre eux via des nœuds dédiés.

Les serveurs sont les machines (ordinateurs) qui hébergent des sites web ou des services en ligne et permettent aux machines clientes (ordinateurs des internautes) d'accéder.
Plats, sans écran et empilés les uns sur les autres dans des baies informatiques, les serveurs sont connectés à Internet via des équipemets réseaux et de télécommunications.

Les datacenters sont des bâtiments très sécurisés dotés de salles à environnement contrôlé (température, hygrométrie, poussières, etc) dans lesquelles sont hébergées et gérées de nombreuses baies informatiques et des équipements réseaux.
Les datacenters, reliés à un câble de fibre optique, permettant des échanges ultrarapides de données en grands nombres. Cette fibre optique est en général enterrée dans le sol, mais on la retrouve aussi au fond des mers et des océans. La majorité du trafic d'Internet passe par des câbles sous-marins intercontinentaux.

A contrario, les serveurs locaux ne sont pas de véritables serveurs car ils présentent une double exception. En effet, ils ne sont pas hébergés dans des datacenters et ce ne sont pas de véritables serveurs mais ce sont les ordinateurs des développeurs.
Les serveurs locaux permettent aux développeurs de créer et de tester, en langage backend (PHP, Node.JS, Python, etc), des sites Web sur leurs propres ordinateurs ; grâce à des outils permettant de simuler le fonctionnement d'un serveur.
Les développeurs peuvent ainsi exécuter les langages backend, sans avoir besoin de transférer les sites sur de véritables serveurs pendant la phase de développement. Les outils les plus connus, pour ce faire, sont MAMP pour Mac, WampServer pour Windows ou encore LAMP pour Linux.

Adresse IP

Chaque ordinateur possède une adresse spécifique pour communiquer avec les autres ordinateurs connectés à Internet, via le protocole de communication dédié : Internet Protocol. L'adresse IP d'un ordinateur se présente sous la forme d'une suite de chiffres de type 205.89.177.26., ce que l'on peut assimiler à un numéro de téléphone.

En théorie l'on peut donc aller sur un site web en tapant directement l'adresse IP du serveur dans la barre d'adresse de son navigateur Web. Mais cela n'est pas très pratique, c'est pourquoi ont été créés les noms d'hôtes tels « google.com » ou « free.fr ».

Nom d'hôte et DNS

Les noms d'hôte ou noms de domaine, tel « google.fr », permettent de retrouver très facilement le nom du service (ou du site Internet) auquel l'on veut accéder.
C'est pour cela qu'a été créé un service d'annuaire : le DNS (Domain Name Service), un service informatique permettant d'associer un nom d'hôte avec une adresse IP.

En conclusion le Domain Name Service peut être vu comme un annuaire géant où l'on retrouve tous les noms d'hôtes (google.fr, openclassrooms.com, etc) avec l'adresse IP associée.

Protocoles

Les protocoles sont des langages de communication pour les ordinateurs et les serveurs. Ces langages permettent aux ordinateurs de dialoguer selon deux familles de protocoles dits de bas niveau et de haut niveau.

Les protocoles de bas niveau sont à la base d'Internet, on les retrouve dans toutes les communications :

  • TCP (Transmission Control Protocol)
  • UDP (User Datagram Protocol)

TCP est un protocole très important qui a été inventé par Vint Cerf, considéré aujourd'hui grâce à cela comme le père d'Internet. Sans protocle TCP il n'y aurait pas de communication réseau... et Tim Berners-Lee n'aurait jamais pu inventer le Web.
Le Transmission Control Protocol est utilisé pour faire beaucoup de choses : charger des pages web, consulter des e-mails, lire des vidéos, écouter de la musique...

Les protocoles de haut niveau sont généralement basés sur TCP ou sur UDP. Ils forment une surcouche qui englobe le protocole TCP.

Les noms des protocoles de haut niveau nous sont plus familiers, notamment certains d'entre eux :

  • HTTP est le protocole qui permet d'échanger des pages web entre le client et le serveur. En plus du langage HTML, Tim Berners-Lee a inventé les bases du protocole HTTP qui permet d'échanger les pages. C'est ce que signifie le "http://" que l'on voit au début des adresses des sites web.
  • HTTPS est issu du protocole HTTP, avec le « S » en plus qui signifie « Secure » ou sécurisé. Les pages sont chiffrées, pour garantir que personne ne peut les lire (par exemple via l'espionnage des données qui transitent dans les câbles sous-marins).
  • FTP permet de réaliser des échanges et des transferts de fichiers via un logiciel dédié à ce protocole.
  • SFTP est issu du protocole FTP, il constitue la version sécurisée d'échange de fichiers.
  • SMTP est le protocole qui permet d'envoyer des e-mails.
  • Etc.

Partie 4 - Identifiez les métiers du web

Développeurs Web

Le développeur est une personne qui conçoit, développe, teste, documente et maintient des applications, des sites Internet ou des logiciels.

Contrairement à ce qu'on peut s'imaginer, le développeur web ne passe pas toute sa journée à écrire des lignes de code. Quelle que soit sa spécialité, il passe beaucoup de temps à tester son code, à chercher à optimiser ses méthodes de développement et à travailler avec ses pairs.
L'un de ses plus grands challenges est d'analyser le besoin initial du client ou de l'utilisateur ; le développeur doit comprendre ce besoin pour le traduire en une ou plusieurs langages de programmation.

Il existe différents métiers dans le développement web, dont voici les principaux :

  • Le développeur front-end travaille sur toute la partie apparente du site, ce que nous voyons lorsque nous allons sur une page web. Il crée l'interface utilisateur de telle sorte qu'elle soit la plus adaptée aux besoins, et la plus simple possible. Il utilise les langages client HTML, CSS et JavaScript.
  • Le développeur back-end travaille au contraire sur toutes les composantes cachées d'un site Web. Il crée l'infrastructure adaptée et met en place les liaisons entre le site et les bases de données. Il connaît bien un ou plusieurs langages de programmation serveur (PHP, Ruby, Python, Java), le langage de requêtage sur bases de données (SQL), ainsi que les frameworks qui lui facilitent la tâche.
  • Le développeur mobile réalise des applications Android ou iOS, parfois même les deux, même si c'est peu fréquent. Il travaille sur toutes les étapes de la conception : depuis le cahier des charges jusqu'à la correction des bugs. Quand on développe sur mobile, on travaille en général aussi bien sur l'interface (front-end) que sur la logique (back-end).
  • Le développeur low-code utilise des plateformes (Kosmos Digital, Microdoft Power Apps, Zoho creator, etc) qui permettent de créer des applications avec peu ou pas de code. L'objectif est d'assembler des blocs fonctionnels existants pour construire rapidement des solutions. Le développeur low-code connaît les principes de la programmation et les besoins métier, mais il écrit très peu de code lui-même.

Product Managers et chefs de projets

C'est pour éviter les désagréments de types : fonctionnalités inadaptées et problèmes de mises à jour qu'existent les Product Owners et les Product Managers.
Le Product Owner veille à ce que chaque détail du produit réponde aux besoins immédiats des utilisateurs et soit correctement implémenté.
Le Product Manager, quant à lui, assure que le produit évolue dans la bonne direction, en anticipant les besoins futurs et en définissant une stratégie à long terme.
Ensemble, ils garantissent que le produit soit à la fois fonctionnel et en phase avec les attentes du marché.

Le quotidien du Product Owner
Très concrètement, le Product Owner doit s'assurer que le produit développé réponde aux besoins des utilisateurs et aux objectifs fixés par le Product Manager. Pour réussir sa mission, le Product Owner va rédiger et prioriser les user stories, des descriptions courtes et précises des fonctionnalités à développer. Il anime les réunions de planification et de revue, en s'assurant que chaque fonctionnalité livrée apporte de la valeur au produit.
Il teste régulièrement les développements pour vérifier qu'ils correspondent bien aux attentes initiales. Ainsi avant de valider une nouvelle fonctionnalité de filtre sur un site e-commerce, il teste différentes combinaisons pour s'assurer que le filtre fonctionne correctement.
Enfin, le Product Owner travaille étroitement avec les développeurs, les designers et les testeurs, veillant à ce que la vision du produit soit bien comprise et respectée par tous.

Le quotidien du Product Manager
Le Product Manager a une vision plus stratégique du produit. Il analyse le marché, identifie les opportunités et définit la roadmap produit à long terme. Pour réussir sa mission, le Product Manager mène des études de marché, analyse les données utilisateurs et surveille les tendances. Ainsi il peut étudier les besoins des consommateurs pour lancer une nouvelle fonctionnalité de paiement mobile sur une application bancaire.
Il élabore une vision globale du produit et s'assure que toutes les parties prenantes partagent cette vision. Il collabore avec les équipes marketing pour définir les stratégies de lancement et avec les équipes de vente pour optimiser la commercialisation. Par exemple, avant le lancement d'un nouveau produit, il planifie une campagne publicitaire et forme l'équipe de vente sur les nouvelles fonctionnalités.
Il est responsable de la rentabilité du produit et de son évolution à long terme. Il s'assure que chaque décision prise contribue à la réussite du produit sur le marché.

Designers UX et UI

L'UX Designer (User Experience Designer) travaille à ce que l'internaute (ou le mobinaute) s'oriente aisément et trouve rapidement et efficacement l'information qu'il est venu chercher. Il fait en sorte que l'utilisation d'un site ou d'une application soit la plus simple possible, que la navigation soit fluide et intuitive ; en clair que l'on prenne plaisir à utiliser le produit.
Très concrètement, l'UX Designer doit s'assurer que le site ou l'application réponde aux attentes et aux besoins de ses utilisateurs, qu'il procure des émotions positives.

Pour réussir sa mission, l'UX Designer va utiliser plusieurs méthodes « centrées utilisateur ». Il va définir le public cible du produit, mener des « recherches utilisateur » via des entretiens, des observations in situ, des analyses de données. Il cherche à comprendre et à analyser les attentes des utilisateurs. Il va créer des prototypes, les tester avec de potentiels futurs utilisateurs... C'est ce qu'on appelle la conception centrée sur l'humain ou Human-Centered Design (HCD).
Le quotidien d'un UX Designer est donc fait de beaucoup d'interactions : l'UX Designer rencontre régulièrement des utilisateurs et doit faire preuve d'empathie pour comprendre leurs besoins et attentes. Aussi, l'UX Designer est en lien avec les équipes techniques, dont les développeurs ! Ces professionnels travaillent tous pour concevoir le produit, mais sous des angles différents et complémentaires.

Quelles différences entre l'UX Designer et l'UI Designer ?

Egalement connu sous le nom de Webdesigner, l'User Interface Designer mobilise des méthodologies très proches de l'UX, c'est à dire centrées sur l'humain. Alors que l'UX Designer s'intéresse à l'expérience globale de l'utilisateur, l'UI Designer va se concentrer sur l'interface. En lien avec les équipes techniques, il donne vie au produit en mixant des éléments graphiques et des éléments fonctionnels, de telle sorte que le produit soit le plus attractif, agréable et accessible possible.
En résumé, l'UI Designer et l'UX Designer travaillent sur des missions qui se recoupent. D'ailleurs dans certaines structures on ne différencie pas les deux métiers et on parle de « Product Designer ». C'est souvent le cas dans les petites entreprises, qui vont chercher des profils généralistes capables de travailler sur toutes les dimensions du design d'un produit.

Testeurs

La Quality Assurance ou QA, est le nom que porte l'équipe de testeurs en charge de s'assurer que les produits développés sont fiables, fonctionnent correctement et offrent une expérience utilisateur optimale.

Les testeurs de logiciel ou équipe QA utilise des processus de test rigoureux, comprenant à la fois des tests manuels et des tests automatisés, pour identifier les défauts et bugs dans les sites ou applications web. En collaborant avec les équipes de développement, l'équipe QA contribue à améliorer la qualité du produit, à réduire les risques et à renforcer la confiance des clients.

Les principaux métiers au sein de l'équipe QA sont les suivants :

  • L'ingénieur QA ou testeur de logiciel est chargé de concevoir, d'exécuter et d'automatiser les tests pour garantir la qualité des logiciels ou des produits. Il identifie les bugs et les problèmes de performance afin de les résoudre avant la mise en production.
  • L'analyste QA assure l'analyse des exigences des clients et les spécifications des produits pour élaborer des plans de test détaillés. Il s'assure que les fonctionnalités développées correspondent aux attentes et aux besoins des utilisateurs.
  • Le responsable QA supervise et coordonne les activités de l'équipe QA. Il définit les stratégies de test, établit les normes de qualité et assure la conformité aux processus et aux exigences de l'entreprise.

Techniciens informatique

Le rôle du technicien informatique au sein d'une entreprise est d'assurer le bon fonctionnement du parc informatique, mais aussi de fournir aux utilisateurs des solutions pour pallier les éventuelles pannes ou difficultés qu'ils rencontrent.

Le poste de technicien informatique est extrêmement polyvalent, il est chargé de différentes missions :

  • Il supervise le matériel informatique tels les ordinateurs et les périphériques, ainsi que les systèmes d'exploitation (Windows, OS X, Linux, etc). Il configure et entretient le parc informatique, gère les stocks et veille à son bon fonctionnement.
  • Il s'assure que tous les postes de travail aient accès aux applications et services nécessaires, en mettant en place l'architecture des réseaux et du WiFi.
  • Le technicien informatique assure le support et le conseil aux utilisateurs, en étant leur interlocuteur privilégié pour tous les problèmes informatiques : de la réception des demandes jusqu'à l'assistance en cas de panne. Il gère les problèmes en fonction de leur urgence et aide les utilisateurs à devenir autonomes. C'est un métier qui exige patience et pédagogie.

Boîte à outils du développeur

  • IDE (Integrated Development Environment) :

    Pour rédiger et mettre en forme un document l'on utilise un traitement de texte (Word, Write, etc), un tableur (Excel, Calc, Google Sheets, etc) ou un logiciel de présentation (PowerPoint, Impress, etc).
    Mais quand l'on veut écrire du code informatique, soit l'on utilise un éditeur de texte) avec des fonctionnalités étendues (Notepad++, Sublime Text, etc), soit l'on utilise un IDE ou Integrated Development Environment.

    Pour accélérer le développement d'un site Internet ou d'un logiciel tout en améliorant sa cohérence et sa qualité, il est nécessaire d'utiliser un outil plus puissant, à savoir un IDE ou « environnement de développement intégré ».
    Tout Integrated Development Environment comporte au minimum un éditeur de code, un débogeur ainsi qu'un compilateur ou éditeur de liens. Des fonctionnalités complémentaires peuvent être ajoutées telles un testeur de code, un moteur de recherche, un outil de contrôle de versions, etc.

    Bien que certains IDE soient dédiés à un langage de programmation particulier, les IDE les plus utilisés tels Visual Studio Code (VSCode), IntelliJ IDEA, Eclipse ou NetBeans sont multilanguages.

  • Github :

    Plateforme Web pour codeurs, Github permet aux développeurs de créer, de stocker, de gérer, de corriger, de partager avec leurs pairs ou de mutualiser du code informatique.
    Pour ce faire, la plateforme utilise le gestionnaire décentralisé de versions de fichiers : Git. Celui-ci permet de travailler de manière collaborative sur du code, sans risquer de détruire les modifications ou les ajouts effectués par d'autres développeurs.

    Pour résumer, Github est un service en ligne d'hébergement de versions successives de code informatique ou de fichiers. Il dispose d'un système de suivi des modifications et de traçabilité des interventions réalisées par chaque codeur.
    GitHub dispose de fonctionnalités de contrôle d'accès, de suivi des bugs, de demandes de fonctionnalités logicielles, de gestion des tâches, d'intégration continue et de wikis.

  • Librairies / bibliothèques :

    Une librairie ou blibliothèque est un ensemble de routines informatiques (fonctions, classes, etc)) déjà codées dans un langage spécifique. Le développeur va alors directement piocher le code nécessaire dans une librairie du langage de programmation qu'il utilise pour un projet.
    Les bibliothèques permettent de simplifier et de rationaliser la programmation. C'est également un gain de temps pour les développeurs, cela leur permet également d'améliorer la qualité de leur code et de gagner du temps tout en bénéficiant du soutien de la communauté de leurs pairs.

    Parmi les très nombreuses librairies disponibles, certains ont été créées pour un système d'exploitation (MacOS, Windows, Unix, etc), un langage de programmation (JavaScript, PHP, Python, etc), un framework (ASP.NET, Flask, Laravel, etc) ou un système de gestion de contenus (Wordpress, Wix, Prestashop, etc).

  • Frameworks :

    Egalement appelé infrastructure logicielle ou cadre d'applications, le framework est un ensemble de composants logiciels structurels, de bibliothèques, de modèles de conception, d'outils et de conventions de codage. Les frameworks permettent ainsi d'accélérer le processus de développement logiciel.
    Le framework se distingue d'une bibliothèque ou d'une librairie par son caractère générique ; il peut d'ailleurs en compter plusieurs. Servant de cadre au développement de progiciels et autres applications, il en modèle l'architecture logicielle.

    Chaque framework est dédié à une plateforme informatique (Eclipse RCP, etc), à un domaine particulier (CubicWeb pour le web sémantique, FxEngine pour le traitement du signal, etc) ou à un langage de programmation (Symfony pour PHP, Django pour JavaScript, .Net pour C# et Visual Basic .NET, etc).
